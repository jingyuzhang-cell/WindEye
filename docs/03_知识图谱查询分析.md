# 03 — 知识图谱查询分析

## 3.1 模块概述

知识图谱查询分析模块是 BiDA-KG 的核心可视化入口，提供四层知识图谱（主体层 → 事件层 → 特征层 → 法规层）的数据统计、多维度检索、层级穿透和图谱可视化功能。

### 3.1.1 路由结构

| 路由 | 页面 | 说明 |
|------|------|------|
| `/knowledge-graph` | GeneralPage | 四层总览，跨层展示 |
| `/knowledge-graph/subject` | SubjectPage | 主体层（企业/自然人/私募） |
| `/knowledge-graph/event` | EventPage | 事件层（事件/时间） |
| `/knowledge-graph/feature` | FeaturePage | 特征层（风险特征/风险因子） |
| `/knowledge-graph/regulation` | RegulationPage | 法规层（法规/法律） |

### 3.1.2 技术栈

- **前端**: React 19 + Ant Design Pro 6 + @antv/g6 v4.8 + Umi Max v4 + TypeScript
- **后端**: FastAPI + Neo4j Python driver + Neo4j Aura
- **构建**: Mako (Rust bundler)

## 3.2 核心功能需求

### 3.2.1 全局数据统计面板

**位置**: 页面顶部
**组件**: 统计卡片 + 详情弹窗

| 指标 | 数据来源 |
|------|---------|
| 总节点数 | GET `/api/v1/graph/summary-stats` → `total_nodes` |
| 总关系数 | GET `/api/v1/graph/summary-stats` → `total_relationships` |
| 当前节点数 | 图谱当前加载的节点数量 |
| 当前关系数 | 图谱当前加载的关系数量 |
| 各层详情 | 四层各自的节点数、节点类型、关系数、关系类型 |

**交互**: 点击"详情"按钮弹出 Modal，展示四层完整统计表格。

### 3.2.2 检索功能

**位置**: 页面左侧/顶部表单区域
**组件**: SearchForm（关键词输入 + 层级选择 + 深度滑块）

| 参数 | 组件 | 说明 |
|------|------|------|
| 关键词 | Input.Search | 按节点名称模糊检索 |
| 检索层级 | Select | 选择限定层级：全部/主体/事件/特征/法规 |
| 穿透深度 | Slider (1-5) | N-hop 关系穿透层数 |

**API**: GET `/api/v1/graph/search-all?q=<keyword>&depth=<depth>&layer=<layer>&limit=200`

**行为**:
- 无关键词时加载全图数据
- 有关键词时从匹配节点出发做 N-hop 扩展
- 搜索结果合并到当前图谱

### 3.2.3 图谱可视化

**位置**: 页面中央主区域
**组件**: G6 Graph (Canvas 渲染器)

**节点视觉规范:**

| 层级 | 节点类型 | 颜色 | 形状 |
|------|---------|------|------|
| 主体层 | COMPANY (企业) | #FFC101 | circle |
| 主体层 | PERSON (自然人) | #1890FF | circle |
| 主体层 | PFCOMPANY (私募公司) | #722ED1 | circle |
| 主体层 | PFUND (私募基金) | #008000 | circle |
| 事件层 | EVENT (事件) | #FF6B6B | rect |
| 事件层 | SUB_EVENT (子事件) | #FF8C94 | rect |
| 事件层 | TIME (时间) | #45B7D1 | diamond |
| 特征层 | RiskFeature (风险特征) | #4CAF50 | ellipse |
| 特征层 | RiskFactor (风险因子) | #9C27B0 | ellipse |
| 法规层 | Action (处置措施) | #00BCD4 | rect |
| 法规层 | Regulation (法规) | #FFC101 | rect |

**关系视觉规范:**

| 关系类型 | 颜色 | 线型 | 说明 |
|---------|------|------|------|
| TRIGGERS | #FF6B6B | solid | 事件触发 |
| REFLECTS | #4CAF50 | solid | 风险体现 |
| COMPLIES_WITH | #00BCD4 | dashed | 合规关联 |
| CAUSE | #FF8C00 | solid | 因果关系 |
| BELONG | #9C27B0 | dotted | 归属关系 |
| INVEST | #1890FF | solid | 投资关系 |
| GUARANTEE | #FF6B6B | dashed | 担保关系 |
| BRANCH | #722ED1 | solid | 分支机构 |
| WORK | #45B7D1 | dotted | 任职关系 |

**布局**: 四层分泳道（Swimlane）垂直排列，同层节点水平分布。支持切换：力导向 / 分层(Dagre) / 环形 / 同心圆。

### 3.2.4 层级穿透功能

**触发**: 双击节点 / 点击"展开"按钮
**API**: GET `/api/v1/graph/expand/{node_id}?depth=1&limit=100`

**行为**:
1. 以选中节点为中心，扩展 N-hop 邻居
2. 新节点/关系通过 `graph.addItem()` 增量合并
3. 已展开节点记录在 `Set<string>` 中，避免重复
4. 展开中显示节点级 loading 动画
5. 支持收起：移除该节点的扩展子图

### 3.2.5 交互功能

| 交互 | 实现 |
|------|------|
| 缩放 | 鼠标滚轮 / 触控板捏合 (0.1x - 5x) |
| 平移 | 拖拽画布空白区域 |
| 节点拖拽 | 拖拽节点调整布局 |
| 节点悬浮 | 高亮节点及其1-hop邻居 |
| 节点点击 | 打开 Drawer 展示节点属性和关联关系 |
| 节点双击 | 以该节点为中心展开1-hop子图 |
| 画布自适应 | 工具栏"适应画布"按钮 |

### 3.2.6 数据导出

| 导出类型 | 触发 | 格式 |
|---------|------|------|
| 图谱截图 | "导出图谱"按钮 | PNG (G6 downloadFullImage) |
| 统计数据 | "导出统计"按钮 | CSV (BOM UTF-8) |

## 3.3 数据结构

### 3.3.1 节点数据

```typescript
interface GraphNode {
  element_id: string;      // Neo4j 元素ID
  labels: string[];         // 标签列表 (e.g. ['Subject', 'COMPANY'])
  properties: Record<string, any>;  // 属性键值对
  // G6 渲染扩展字段
  id: string;              // = element_id
  typeKey: string;         // 前端类型键 (COMPANY, EVENT, etc.)
  layer: number;           // 层级索引 0-3
  color: string;           // 节点颜色
  name: string;            // 显示名称
  x?: number;              // 布局X坐标
  y?: number;              // 布局Y坐标
}
```

### 3.3.2 关系数据

```typescript
interface GraphEdge {
  element_id: string;      // Neo4j 元素ID
  type: string;            // 关系类型 (INVEST, GUARANTEE, etc.)
  startNodeElementId: string;  // 源节点ID
  endNodeElementId: string;    // 目标节点ID
  properties: Record<string, any>;
  // G6 渲染扩展字段
  source: string;          // = startNodeElementId
  target: string;          // = endNodeElementId
  label: string;           // 显示标签
  style: { stroke: string; lineWidth: number; endArrow: boolean };
}
```

### 3.3.3 统计数据

```typescript
interface SummaryStats {
  total_nodes: number;
  total_relationships: number;
  layers: LayerStats[];
}

interface LayerStats {
  layer: string;           // "Subject" | "Event" | "Feature" | "Regulation"
  layer_label: string;     // "主体层" | "事件层" | "特征层" | "法规层"
  node_count: number;
  node_type_count: number;
  node_types: string[];
  rel_count: number;
  rel_type_count: number;
  rel_types: string[];
}
```

## 3.5 性能目标

| 指标 | 目标 | 策略 |
|------|------|------|
| 节点数 | 1000+ | `changeData()` 代替 destroy/recreate |
| 关系数 | 2000+ | Canvas 渲染器，禁用大图动画 |
| 首屏加载 | <3s | 限制初始 limit=100 |
| 搜索响应 | <2s | 后端 limit=200，前端增量合并 |
| 展开响应 | <1s | 1-hop 子图查询 + `addItem()` 增量 |
| 导出PNG | <5s | G6 内置 downloadFullImage |

## 3.6 非功能性需求

### 3.6.1 加载状态
- 全图加载: 全屏 Spin + "加载图谱数据中..."
- 搜索: 搜索按钮 loading + 保留上一次结果
- 展开: 节点级 loading 动画

### 3.6.2 错误处理
- 网络错误: 红色 Alert + "后端服务连接失败，请检查服务是否启动"
- 空数据: Empty 组件 + "未找到匹配的节点"
- 请求超时: 30s 超时 + 自动重试

### 3.6.3 可访问性
- 节点支持键盘导航 (Tab 切换)
- 颜色对比度满足 WCAG AA
- 关键操作提供 Tooltip

### 3.6.4 代码质量
- TypeScript 严格模式
- 模块化设计：统计/搜索/图谱/导出独立组件
- DRY 原则：四层页面共享 LayerGraphPage 组件

## 3.7 系统架构定位

图谱查询分析位于业务应用层，是用户与知识图谱交互的主界面：

```
┌──────────────────────────────────────────────────────────────┐
│                     业务应用层 (Application)                   │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────────────┐  │
│  │ 知识图谱检索  │ │ 协同治理问答  │ │  风险可视化看板       │  │
│  │ (GeneralPage) │ │ (KnowledgeQA) │ │  (G6 图分析)         │  │
│  └──────────────┘ └──────────────┘ └──────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────┐
│                  四层知识图谱 (Neo4j)                          │
│    主体层(Subject) → 事件层(Event) → 特征层(Feature) → 法规层  │
└──────────────────────────────────────────────────────────────┘
```

**技术选型：**
- 图渲染引擎：AntV G6 v4，支持自定义节点/边、布局算法、交互插件
- 3D/大规模图：react-force-graph-2d，力导向布局的大型图渲染
- 节点样式：自定义 G6 Node + 图片节点，支持企业 Logo、风险图标
- 数据流：WebSocket (实时更新) + REST (查询)
