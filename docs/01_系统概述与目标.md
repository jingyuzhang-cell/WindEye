# 01 — 系统概述与建设目标

## 1.1 业务痛点

传统风控与治理手段面临以下核心困境：

| 痛点 | 表现 | 根因 |
|------|------|------|
| **数据孤岛** | 工商、司法、舆情、内部数据分散在不同系统中，无法关联分析 | 缺乏统一的数据融合与实体对齐机制 |
| **隐性风险传递** | A 公司暴雷 → 担保链 → 高管交叉任职 → 风险扩散至表面上无关的实体 | 缺乏多跳关系推理能力 |
| **规则滞后** | 基于静态规则的风控模型无法应对新型欺诈模式 | 规则引擎不具备语义理解和动态推理能力 |
| **协同低效** | 风险发现→分析→处置→反馈各环节割裂，依赖人工传递信息 | 缺乏自动化协同工作流 |
| **报告成本高** | 一份深度风险报告需要分析师数小时才能完成 | 缺乏自动化的信息聚合与报告生成能力 |

## 1.2 建设目标

构建基于 **"数据采集 → 四层知识图谱 → 多智能体协同推理"** 的智能化风险协同治理平台。

| 目标维度 | 具体目标 |
|----------|---------|
| **看得见** | 将分散的多源异构数据融合为一张统一的知识图谱，支持多跳关系可视化探查 |
| **想得透** | 利用 LLM 驱动的多智能体系统，自动推理风险传递路径，发现隐藏的关联模式 |
| **管得住** | 自动生成协同治理社区报告，沉淀风险主体、群体发现、传导路径和治理建议 |

## 1.3 核心价值

- **效率提升：** 风险分析报告生成时间从数小时压缩到分钟级
- **覆盖广度：** 从单点分析扩展为 N 跳全图分析，不遗漏间接关联风险
- **决策质量：** 多 Agent 协同（检索 + 分析 + 合规）交叉验证，降低单一模型误判率
- **知识沉淀：** 每次分析、处置、反馈都回流至知识图谱，持续增强系统认知能力

## 1.4 典型业务场景

> **场景：企业信贷风险传导预警**
>
> 系统监测到"公司A"新发生"被法院列为被执行人"事件。自动触发多智能体协同分析：
>
> 1. **检索 Agent** 查询 Neo4j，提取公司 A 的 3 层关联网络（股东、对外投资、高管任职、担保关系）
> 2. **分析 Agent** 发现公司 A 的法人代表同时是公司 B 的大股东，且公司 B 近期有大额资金转出事件
> 3. **合规 Agent** 匹配法规库，认定存在"恶意转移资产"的高嫌疑，引用相关法条
> 4. **汇报 Agent** 撰写《关于公司 A 及其关联方资产转移风险预警报告》，推送至治理报告面板，形成治理建议和持续观察清单
>
> 整个流程从事件触发到报告生成，目标耗时 < 60 秒。
