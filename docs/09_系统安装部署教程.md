## 2. 基础环境安装

### 2.1 Python

**Windows：**
1. 访问 [python.org/downloads](https://www.python.org/downloads/)
2. 下载 Python 3.11+ 安装包（64-bit）
3. **勾选 "Add Python to PATH"**
4. 点击 Install Now

验证安装：
```powershell
python --version
pip --version
```

### 2.2 Node.js

**Windows：**
1. 访问 [nodejs.org](https://nodejs.org/)
2. 下载 LTS 版本（20.x）
3. 运行安装包，使用默认选项

验证安装：
```powershell
node --version
npm --version
```

### 2.3 Git

**Windows：**
1. 访问 [git-scm.com/download/win](https://git-scm.com/download/win)
2. 下载 64-bit 安装包
3. 安装时选择 "Use Git from the command line"

验证安装：
```powershell
git --version
```

### 2.4 Google Chrome

**Windows：**
1. 访问 [google.com/chrome](https://www.google.com/chrome/)
2. 下载并安装最新版 Chrome

> Chrome 用于 Selenium WebDriver 驱动的网页爬虫。系统会自动匹配 Chrome 版本下载对应的 ChromeDriver。

## 3. Neo4j 图数据库安装与配置

### 3.1 安装 Neo4j Community Edition

**Windows：**
1. 访问 [neo4j.com/download](https://neo4j.com/download/)
2. 选择 **Neo4j Community Edition**
3. 下载 Windows 安装包（.exe）
4. 运行安装包，按向导完成安装

### 3.2 创建数据库

1. 启动 Neo4j Desktop
2. 点击 "New" → "Create project"
3. 在项目中点击 "Add" → "Local DBMS"
4. 设置数据库名称（如 `WindEye`）
5. 设置密码（记住此密码，后续配置需要）
6. 版本选择 5.x
7. 点击 "Create"

### 3.3 启动数据库

1. 在 Neo4j Desktop 中点击数据库旁的 "Start"
2. 等待状态变为绿色 "Running"
3. 默认连接信息：
   - Bolt URI: `bolt://localhost:7687`
   - HTTP URI: `http://localhost:7474`
   - 用户名: `neo4j`
   - 密码: 步骤 3.2 中设置的密码

### 3.4 安装 APOC 插件（可选，用于高级图算法）

1. 在 Neo4j Desktop 中选择数据库
2. 点击 "Plugins" 标签
3. 找到 **APOC** 并点击 "Install"
4. 重启数据库

## 4. MySQL 安装与初始化

### 4.1 安装 MySQL

安装 MySQL Community Server，记住安装时设置的 `root` 密码，并确认服务已启动：

```powershell
mysql --version
mysql --host=127.0.0.1 --port=3306 --user=root -p
```

### 4.2 准备初始化用户权限数据库

初始化工具随后端源码提供。请先完成第 5 节的 Python 依赖安装，再执行：

```powershell
python -m db.seed
```

执行前确保已在 `backend/.env` 中填写了 `MYSQL_PASSWORD`。脚本自动创建：

- 8 张核心表（如不存在则自动建表）
- 管理员账号 `admin` / `admin`
- 4 个角色（管理员/分析师/审核员/只读用户）
- 18 项权限 + 角色权限映射

脚本可重复执行，已存在的数据会自动跳过。

详细表结构见 [08_用户权限与审计日志.md](08_用户权限与审计日志.md) 和
[backend/db/MYSQL_SETUP.md](../backend/db/MYSQL_SETUP.md)。

## 5. 后端环境配置

### 5.1 创建虚拟环境

```powershell
# 进入项目目录
git clone https://github.com/chensen-tech/WindEye.git
cd WindEye\backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
venv\Scripts\activate
```

激活成功后，命令行前会出现 `(venv)` 标识。

### 5.2 安装 Python 依赖

```powershell
# 确保在 backend/ 目录且虚拟环境已激活
pip install -r requirements.txt
```

安装完成后初始化 MySQL：

```powershell
python -m db.seed
```

### 5.3 下载 spaCy 中文模型

本地 NER 实体提取依赖 spaCy：

```powershell
python -m spacy download zh_core_web_sm
```

如果下载失败，可使用镜像：
```powershell
pip install zh-core-web-sm -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 7. 环境变量配置

### 7.1 创建配置文件

在 `backend/` 目录下创建 `.env` 文件：

```powershell
# 从 frontend 返回 backend
cd ..\backend

# 从安全模板复制
Copy-Item .env.example .env
```

### 7.2 编辑 .env 文件

用文本编辑器（VS Code / Notepad++）打开 `.env`，填入以下内容：

```env
# ============================================================
# Neo4j 数据库连接（必须）
# ============================================================
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=你的Neo4j密码
NEO4J_DATABASE=neo4j

# ============================================================
# MySQL、认证与审计（必须）
# ============================================================
MYSQL_ENABLED=true
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=你的MySQL密码
MYSQL_DATABASE=user
MYSQL_POOL_SIZE=5
MYSQL_POOL_MAX=20

AUTH_MODE=enforce
JWT_SECRET=替换为至少32位的随机密钥
JWT_ALGORITHM=HS256
JWT_ACCESS_EXPIRE_MINUTES=120
JWT_REFRESH_EXPIRE_DAYS=7

# ============================================================
# Redis（推荐，用于 Token 黑名单和权限缓存）
# ============================================================
REDIS_ENABLED=false
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0
REDIS_PASSWORD=

# ============================================================
# 审计日志
# ============================================================
AUDIT_API_LOG_ENABLED=true
AUDIT_OPERATION_LOG_ENABLED=true
LOG_RETENTION_DAYS=180
LOG_SLOW_REQUEST_MS=3000

# ============================================================
# LLM API 配置（DeepSeek，NER/分析需要）
# ============================================================
LLM_API_KEY=你的DeepSeek_API_Key
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat

# ============================================================
# Dify 工作流 API（事件/特征/法规三层实体抽取）
# ============================================================
# 主体提取使用本地 NER，不需要 DIFY_SUBJECT_API_KEY
DIFY_API_KEY=
DIFY_EVENT_API_KEY=
DIFY_FEATURE_API_KEY=
DIFY_REGULATION_API_KEY=
DIFY_BASE_URL=https://api.dify.ai

# ============================================================
# 知识图谱数据集与 NER
# ============================================================
KG_DATASET=finance
# rule | spacy | llm | hybrid（推荐 hybrid: 规则 + LLM 联合）
NER_MODEL=hybrid
SPACY_MODEL=zh_core_web_sm

# ============================================================
# 爬虫与 ETL
# ============================================================
# true = 演示模式（模拟数据，无需 Chrome）
# false = 真实模式（使用 Chrome WebDriver）
CRAWL_DEMO_MODE=true
CRAWL_SOURCE_TIMEOUT_SECONDS=15
ETL_BATCH_SIZE=100
ETL_MAX_RETRIES=3
```

生成 JWT 密钥：

```powershell
python -c "import secrets; print(secrets.token_hex(48))"
```

> **最小运行**：必须填写 `NEO4J_PASSWORD`、`MYSQL_PASSWORD` 和 `JWT_SECRET`。
> **开启 LLM 功能**：还需 `LLM_API_KEY`（NER hybrid 模式、风险分析）。
> **开启 Dify 抽取**：还需 `DIFY_EVENT_API_KEY`、`DIFY_FEATURE_API_KEY`、`DIFY_REGULATION_API_KEY`。
> Redis 可选。真实 `.env` 已被 Git 忽略，禁止提交。

### 7.3 获取 API Key

| API | 获取地址 | 用途 |
|-----|----------|------|
| DeepSeek | [platform.deepseek.com](https://platform.deepseek.com/) | LLM 推理和 NER |
| Dify | [cloud.dify.ai](https://cloud.dify.ai/) 或自部署 | 四层实体抽取工作流 |

## 8. 可选组件安装

### 8.1 Tesseract OCR（PDF 扫描件文字识别）

**Windows：**
1. 访问 [github.com/UB-Mannheim/tesseract/wiki](https://github.com/UB-Mannheim/tesseract/wiki)
2. 下载 `tesseract-ocr-w64-setup-5.x.x.exe`
3. 安装时勾选中文语言包 `Chinese (Simplified)`
4. 将 Tesseract 安装路径加入系统 PATH（默认 `C:\Program Files\Tesseract-OCR\`）

安装 Python 绑定：
```powershell
pip install pytesseract Pillow
```

> 如果不使用 OCR 功能（仅处理文字型 PDF），可跳过此步骤。

### 8.2 PyMuPDF（PDF 文本提取，已包含在 requirements.txt）

PyMuPDF 已在 `requirements.txt` 中声明，步骤 5.2 已完成安装。如需手动安装：
```powershell
pip install PyMuPDF
```

### 8.3 Dify 工作流导入（事件/特征/法规三层抽取）

如果使用 Dify 进行事件层、特征层、法规层的实体关系抽取，需要将 YML 工作流导入 Dify 平台：

1. 登录 Dify 平台（[cloud.dify.ai](https://cloud.dify.ai/) 或自部署实例）
2. 进入"工作室" → "导入应用"
3. 导入 `backend/data_collection/dify/` 下的 YML 文件：
   - `金融风险事理图谱完整的流程.yml` — 事理图谱完整抽取
   - `风险特征图谱构建.yml` — 风险特征抽取
   - `政策法规图谱构建-完整.yml` — 法规实体抽取
   - `法规dify/` 下的按风险类型分类的法规抽取工作流
4. 每个导入后的 Dify App 生成一个 API Key
5. 将各 API Key 填入 `backend/.env` 中对应的 `DIFY_*_API_KEY`

> 主体提取使用本地 NER，不需要 Dify 的 Subject API Key。

打开浏览器访问 `http://localhost:8000/docs`，应看到 FastAPI 自动生成的 Swagger 文档页面。

## 11. 常见问题

### Q1: `pip install` 报错或太慢

使用国内镜像：
```powershell
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q2: Neo4j 连接失败

```
neo4j.exceptions.ServiceUnavailable: Unable to retrieve routing information
```

**解决：**
- 确认 Neo4j Desktop 中数据库状态为 "Running"
- 检查 `backend/.env` 中 `NEO4J_URI` 和 `NEO4J_PASSWORD` 是否正确
- 尝试在浏览器中访问 `http://localhost:7474` 验证 Neo4j 是否运行

### Q3: spaCy 模型下载失败

```powershell
# 手动下载
python -m spacy download zh_core_web_sm

# 或使用直接 pip 安装
pip install https://github.com/explosion/spacy-models/releases/download/zh_core_web_sm-3.8.0/zh_core_web_sm-3.8.0-py3-none-any.whl
```

如果需要更小的模型，可使用 `zh_core_web_md` 或 `zh_core_web_lg`。

### Q4: ChromeDriver 不匹配

```
selenium.common.exceptions.SessionNotCreatedException: This version of ChromeDriver only supports Chrome version XXX
```

**解决：**
- 更新 Chrome 到最新版
- 或手动下载匹配的 ChromeDriver：[chromedriver.chromium.org](https://chromedriver.chromium.org/)
- 或将 `CRAWL_DEMO_MODE=true` 使用演示模式跳过爬虫

### Q5: npm install 失败

```powershell
# 清理缓存后重试
npm cache clean --force
npm install

# 或使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### Q6: 端口被占用

```powershell
# 查看端口占用
netstat -ano | findstr :8000
netstat -ano | findstr :8001

# 修改后端端口
python -m uvicorn main:app --host 0.0.0.0 --port 8002 --reload

# 修改前端端口
$env:PORT = '8003'; npm run dev
```

### Q7: Windows 执行脚本权限受限

```powershell
# 以管理员身份运行 PowerShell，执行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q8: 用户管理接口返回 500 或提示 MySQL 未启用

确认 `backend/.env` 中以下配置已填写，并重启后端：

```env
MYSQL_ENABLED=true
MYSQL_PASSWORD=你的MySQL密码
MYSQL_DATABASE=user
AUTH_MODE=enforce
JWT_SECRET=至少32位随机密钥
```

然后重新执行：

```powershell
cd backend
python -m db.seed
```
