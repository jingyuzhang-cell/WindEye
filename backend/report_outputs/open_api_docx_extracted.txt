开放API接口文档
RESTful API接口一般按这几个部分写：请求方法+接口路径+接口说明+请求参数+返回结果+示例
文档项 | 内容
版本 | v1.1
适用模块 | 模块二：知识图谱检索分析；模块三：多元协同治理
接口风格 | RESTful API；请求与响应均使用JSON；大对象建议支持summary/full返回模式
核心链路 | 关键词检索→N跳子图展开→风险主体群体发现→风险传导路径→协同治理社区报告
主要调用方 | 四层图谱展示、协同治理管线、外部业务系统
设计原则 | 以Neo4j原生Node/Relationship数据模型为准；业务结果与前端渲染字段单独定义，不混入基础图谱结构
1.接口总览
序号 | 接口名称 | 方法与路径 | 输入核心 | 输出核心 | 用途
1 | 跨层关键词搜索 | POST /api/v1/graph/search-all | query、layer、depth、limit、type | matchedNodes、nodes、edges、summary | 在主体层、事件层、特征层、法规层中检索关键词，返回匹配节点及其局部关联子图。
2 | N跳展开子图 | POST /api/v1/graph/expand/{node_id} | nodeId、depth、limit、relationWhitelist | centerNode、nodes、edges、subgraph、summary | 以指定节点为中心展开N跳关联网络，用于前端交互展开、深度关联分析和模块三治理分析前置子图构建。
3 | 风险主体群体发现 | POST /api/v1/governance/community-discovery | seedNames / seedIds、maxHop、method、communityMode | seedNodes、subgraph、communities、seedCommunityId、entityCommunityMap、communityGraph | 从风险主体出发抽取k-hop 子图并发现主体所在群体，为前端群体图和后续风险分析提供基础数据。
4 | 风险传导路径分析 | POST /api/v1/governance/risk-paths | seedNames / seedIds、maxHop、maxPathLength、riskRelationWhitelist | riskPaths、communityRiskPaths、viewModel、summary | 识别风险主体到公司、人员、事件、法规等节点的风险传导路径，生成可评分、可解释、可高亮的路径结果。
5 | 协同治理社区报告 | POST /api/v1/governance/compliance-report | query、seedNodes、subgraph、communities、riskPaths、anomalyFindings | compliance、complianceIndicators、governance、report、viewModel | 基于风险路径和证据子图进行合规匹配、指标评分、治理动作生成和报告输出。
2.通用约定
结点层级 | 结点类型
Subject | "Subject", "COMPANY", "PERSON", "PFCOMPANY", "PFUND", "SECURITY","PartyWithResponsibility", "Actor", "Account"
Event | "Event", "EVENT", "Means"
Feature | "Feature", "AdvantageHolder", "Influence","DisadvantageHolder", "Advantage"
Regulation | "Regulation", "Law", "Action", "Chapter"
项目 | 约定
Base URL | http://{host}:{port}，本地开发示例：http://127.0.0.1:8000
Content-Type | application/json;charset=utf-8
命名规范 | HTTP路径使用kebab-case；JSON字段使用camelCase。
字符编码 | UTF-8，支持中文实体名称、中文关系与中文报告内容。
traceId | 所有响应建议返回traceId，便于前后端与后端日志排查。
success | 业务是否成功。success=false时必须返回errorCode/message或error字段。
Response Mode | 建议支持full/summary。full返回完整可视化数据；summary返回摘要，适合外部系统轻量调用。
节点ID优先级 | seedIds优先级高于seedNames；若传入seedIds，则优先按图谱elementId或内部节点ID定位。
通用响应字段 | 类型 | 说明
success | boolean | 请求是否成功。
apiVersion | string | 接口版本，例如v1。建议开放给外部系统时返回。
traceId | string | 链路追踪ID，例如trc-20260605-001。
elapsedMs | number | 接口耗时，单位毫秒。可选。
summary | object | 摘要统计。不同接口内字段略有差异。
warnings | string[] | 非致命异常或降级信息。
errorCode | string | 错误码。success=false时建议返回。
message/error | string | 错误说明。

properties | object | 原始属性或扩展属性，可选。

4. 知识图谱检索与子图扩展API
4.1 接口定义
项目 | 内容
接口名称 | 跨层关键词搜索API
HTTP方法 | POST
路径 | /api/v1/graph/search-all
核心作用 | 输入关键词，在主体层、事件层、特征层、法规层中检索匹配节点，并以匹配节点为中心进行多跳扩展，输出匹配节点、关联子图和三元组JSON。适用于企业、人员、事件、风险特征、法规等跨层图谱检索。
4.2 Request
字段 | 类型 | 必填 | 默认值 | 说明
query | string | 是 |  | 搜索关键词，例如“徐工集团工程机械股份有限公司”“鑫达”“担保风险”。
layer | string | 否 | all | 搜索层级：all/Subject/Event/Feature/Regulation。
depth | number | 否 | 1 | 匹配节点周边展开深度。
limit | number | 否 | 50 | 最大返回节点数。
type | string | 否 | all | 节点类型过滤，例如COMPANY、PERSON、EVENT、Law。
relationWhitelist | string[] | 否 | [] | 允许返回的关系类型。为空时使用默认关系范围。
includeProperties | boolean | 否 | true | 是否返回节点和关系属性。
outputFormat | string | 否 | both | 输出格式：subgraph/triples/both。
deduplicate | boolean | 否 | true | 是否对节点、边和三元组去重。
4.3 Request示例
{  "query": "徐工集团工程机械股份有限公司",  "layer": "all",  "depth": 1,  "limit": 50,  "type": "COMPANY",  "relationWhitelist": [    "INVEST",    "WORK",    "GUARANTEE",    "CONTROLLER",    "SUE",    "JOINDER",    "PARTICIPATE_IN",    "MENTION",    "TRIGGERS",    "REFLECTS",    "COMPLIES_WITH"  ],  "includeProperties": true}
4.4 Response
字段 | 类型 | 说明
success | boolean | 是否成功。
traceId | string | 链路追踪ID。
matchedNodes | Node[] | 直接命中的节点列表。
nodes | Node[] | 返回子图中的节点数组。
edges | Edge[] | 返回子图中的关系数组。
summary | object | 摘要统计，包括命中节点数、节点总数、关系总数、涉及层级、关系类型等。
warnings | string[] | 非致命异常或降级信息。
triples | object | 根据edges生成的三元组JSON。
triples字段三元组JSON结构：
字段 | 类型 | 说明
headId | string | 头实体节点 ID。
head | string | 头实体名称。
headLabels | string[] | 头实体标签。
relation | string | 关系类型，例如 INVEST、WORK、SUE。
relationId | string | 关系 ID。
tailId | string | 尾实体节点 ID。
tail | string | 尾实体名称。
tailLabels | string[] | 尾实体标签。
properties | object | 关系属性。

5. N跳展开子图API
项目 | 内容
接口名称 | N跳展开子图API
HTTP方法 | POST
路径 | /api/v1/graph/expand/{node_id}
核心作用 | 输入一个已知图谱节点ID，围绕该节点展开N跳关联子图，返回目标节点周边的多跳节点和关系。适用于图谱交互展开、风险主体深度分析、群体发现前置子图抽取。
5.1 Request
路径参数 | 类型 | 必填 | 说明
node_id | string | 是 | 需要展开的中心节点ID，通常为Neo4j elementId或系统内部节点ID。
Body字段 | 类型 | 必填 | 默认值 | 说明
depth | number | 否 | 2 | 展开深度，即N跳。
limit | number | 否 | 100 | 最大返回节点数。
relationWhitelist | string[] | 否 | [] | 允许展开的关系类型。为空时使用默认关系范围。
layerWhitelist | string[] | 否 | [] | 允许返回的层级，如Subject、Event、Feature、Regulation。
includeCrossLayer | boolean | 否 | true | 是否包含跨层关系。
includeProperties | boolean | 否 | true | 是否返回节点和关系属性。
5.2 Request 示例
{  "depth": 2,  "limit": 100,  "relationWhitelist": [    "INVEST",    "WORK",    "GUARANTEE",    "CONTROLLER",    "SUE",    "JOINDER",    "PARTICIPATE_IN",    "MENTION",    "TRIGGERS",    "REFLECTS",    "COMPLIES_WITH"  ],  "layerWhitelist": [    "Subject",    "Event",    "Feature",    "Regulation"  ],  "includeCrossLayer": true,  "includeProperties": true}
5.3 Response
字段 | 类型 | 说明
success | boolean | 是否成功。
traceId | string | 链路追踪 ID。
centerNode | Node | 中心节点对象。
nodes | Node[] | N 跳范围内的节点数组。
edges | Edge[] | N 跳范围内的关系数组。
subgraph | Subgraph | 标准子图对象。
summary | object | 摘要统计，包括中心节点 ID、展开深度、节点数量、关系数量、关系类型统计等。
warnings | string[] | 非致命异常或降级信息。
5.4 Response 示例
{  "success": true,  "traceId": "trc-20260605-graph-expand-001",  "centerNode": {    "id": "company_001",    "labels": [      "COMPANY",      "Subject"    ],    "properties": {      "name": "徐工集团工程机械股份有限公司"    }  },  "nodes": [],  "edges": [],  "subgraph": {    "nodeCount": 76,    "edgeCount": 119,    "nodes": [],    "edges": []  },  "summary": {    "centerNodeId": "company_001",    "depth": 2,    "nodeCount": 76,    "edgeCount": 119,    "relationTypes": [      "INVEST",      "WORK",      "SUE",      "JOINDER",      "MENTION"    ]  },  "warnings": []}
6. 风险主体群体发现API
6.1 接口定义
项目 | 内容
接口名称 | 风险主体群体发现API
HTTP方法 | POST
路径 | /api/v1/governance/community-discovery
核心作用 | 输入风险主体名称或节点ID，从知识图谱中抽取k-hop Ego Network，进行WCC/Louvain/HGT+Graph K-means群体发现，输出主体所在群体、全部社区、实体-社区映射和前端可视化数据。
6.2 Request
字段 | 类型 | 必填 | 默认值 | 说明
seedNames | string[] | 条件必填 | [] | 风险主体名称。seedNames 与 seedIds 至少一项非空；若前置 search-all 已确认节点，建议只传 seedIds，避免名称模糊解析带出多种子。
seedIds | string[] | 条件必填 | [] | 图谱节点 ID，优先级高于 seedNames；确认节点后的风险传导分析推荐使用该字段。
autoSelectSeeds | boolean | 否 | false | 是否自动从候选风险主体中选择高风险主体作为种子节点。适用于用户输入事件、文件或宽泛主体后，由系统自动筛选高风险主体。
topKSeeds | number | 否 | 5 | 自动选择前 K 个高风险主体作为种子节点。仅当 autoSelectSeeds=true 时生效。
seedSelectionMode | string | 否 | risk_score | 种子选择方式，可选 risk_score、degree、event_count、bridge_score。
riskConstraints | object | 否 | {} | 控制扩展风险主体时使用哪些约束，包括主体关系、事件关系、风险特征关系和法规关系。
riskConstraints.includeSubjectRelations | boolean | 否 | true | 是否纳入主体层关系，如 INVEST、WORK、GUARANTEE、CONTROLLER。
riskConstraints.includeEventRelations | boolean | 否 | true | 是否纳入事件层关系，如 SUE、JOINDER、PARTICIPATE_IN、MENTION。
riskConstraints.includeFeatureRelations | boolean | 否 | true | 是否纳入风险特征关系，如 TRIGGERS、REFLECTS、CAUSE。
riskConstraints.includeRegulationRelations | boolean | 否 | false | 是否纳入法规映射关系，如 COMPLIES_WITH。
maxHop | number | 否 | 3 | k-hop 子图扩展深度。
method | string | 否 | auto | 群体发现算法：auto / wcc / louvain / hgt_gkmeans。
communityMode | string | 否 | expanded | 子图模式：ego / expanded / global_seed。
minCommunitySize | number | 否 | 2 | 最小社区规模。
pathLimit | number | 否 | 5000 | 子图路径数量上限。
maxNodes | number | 否 | 1000 | 最大返回节点数，避免前端过载。
relationWhitelist | string[] | 否 | [] | 允许扩展的关系类型。为空时使用后端默认关系白名单。
responseMode | string | 否 | full | 返回模式：summary / full。
includeRawSubgraph | boolean | 否 | true | 是否返回原始 subgraph。
includeCommunityGraph | boolean | 否 | true | 是否返回前端可直接绘制的 communityGraph。
includeHgtEmbedding | boolean | 否 | false | 是否返回 HGT 节点向量摘要，默认不返回。

"鑫达投资管理有限公司"
],

],

"includeRawSubgraph": true,
"includeCommunityGraph": true,
"includeHgtEmbedding": false
}
6.4 Response
字段 | 类型 | 说明
success | boolean | 是否成功。
apiVersion | string | 接口版本，例如 v1。
traceId | string | 链路追踪 ID。
elapsedMs | number | 接口耗时，单位毫秒。
selectedMethod | string | 实际采用的社区发现算法。
fallbackReason | string | null | 算法降级原因。
seedNodes | Node[] | 已解析的风险主体节点。若 autoSelectSeeds=true，则为系统最终选中的种子节点。
candidateSeeds | object[] | 候选风险主体列表。系统根据风险事件、主体关系、风险特征、法规约束等计算候选主体风险分。
selectedSeedIds | string[] | 最终用于群体发现的种子节点 ID 列表。
seedSelection | object | 种子选择信息，包括选择模式、Top-K 数量和最终选中数量。
summary | object | 摘要统计，包括 seedNodeCount、nodeCount、edgeCount、communityCount、seedCommunityId 等。
communities | Community[] | 社区列表。
entityCommunityMap | object | 实体到社区的映射，key 必须是 node_id。
communityEdges | object[] | 社区之间的聚合边。
communityGraph | object | 前端默认展示用的社区聚合图。
subgraph | Subgraph | k-hop 原始关联子图。
connectedSubgraph | Subgraph | 最大连通子图。
visualization | object | 前端展示建议，例如 defaultView、suggestedLayout、highlightCommunityId。
warnings | string[] | 非致命异常或降级信息。
6.4.1 Response Mode 说明
community-discovery 支持 summary/full 两种返回模式。接口始终返回 success、apiVersion、traceId、elapsedMs、selectedMethod、fallbackReason、seedNodes、candidateSeeds、selectedSeedIds、seedSelection、summary、warnings。
模式或开关 | 返回字段说明
responseMode=summary | 仅返回通用字段和 summary，不返回 communities、entityCommunityMap、communityGraph、subgraph、connectedSubgraph、visualization。适合列表页、连通性检测和轻量预览。
responseMode=full | 在通用字段基础上返回 communities、entityCommunityMap、communityEdges、communityGraph、connectedSubgraph、visualization，并根据开关返回 subgraph。适合群体图展示、风险传导和社区报告生成。
includeRawSubgraph=false | full 模式下 subgraph 返回 null；connectedSubgraph 仍用于社区发现结果解释。
includeCommunityGraph=false | full 模式下 communityGraph 返回空对象，communityEdges 返回空数组。
业务链路建议：用户输入“徐工集团”等模糊关键词时，先调用 search-all 展示候选节点；用户确认“徐工集团工程机械股份有限公司”后，后续 community-discovery / risk-paths / compliance-report 应优先传 seedIds，而不是再次依赖 seedNames 模糊解析。

Score=RelationRisk×0.40+NodeRisk×0.25+TargetType×0.15+CommunityCrossing×0.10+PathLength×0.10

NodeRisk | 25% | 路径节点风险均值：high=100，medium=60，low=30，unknown=20。
TargetType | 15% | 终点Event/Regulation/Law/Action为100；RiskFeature为70；Person为50；Company为30；其他为20。
CommunityCrossing | 10% | 跨越2个及以上社区边界为100；跨越1个为50；单社区为0。
PathLength | 10% | 路径长度1为100；2为80；3为50；4及以上为30。

>=80 | high
60–79 | medium
<60 | low

附录：图谱查询上限与智能剪枝策略
本节适用于 POST /api/v1/graph/search-all 与 POST /api/v1/graph/expand/{node_id}。当 depth 大于 1 或中心节点连接大量银行、基金、证券、监管机构、时间、公共法规等高出度节点时，接口不应仅依赖 limit/nodeLimit 进行结果截断，而应在遍历阶段启用度数感知与证据优先剪枝，尽量返回对风险传导、群体发现和社区报告更有效的图谱。
A.1 剪枝原则
• 中心节点必须保留。
• 一跳强关系节点优先保留，包括 GUARANTEE、CONTROLLER、CONTROL、INVEST、CAUSE、TRIGGERS。
• Event、Feature、Regulation 证据层优先保留，并优先形成可解释风险链路。
• 一跳节点度数超过 maxExpandDegree 时，默认保留该节点及一跳关系，但不再将其加入下一跳 frontier。
• BANK、PFUND、PFCOMPANY、SECURITY、REGULATOR、EXCHANGE、TIME、Law、Regulation 等标签默认视为高出度低区分度候选。
• limit/nodeLimit 仍作为安全上限；若智能剪枝后仍达到上限，响应必须通过 warnings 与 summary.pruning 说明。
A.2 新增 Request 字段
字段 | 类型 | 默认值 | 说明
traversalMode | string | bfs | 遍历模式：bfs/cascade。治理链路建议使用 cascade。
prunePolicy | string | degree_aware | 剪枝策略：none/degree_aware/evidence_first。
maxExpandDegree | number | 200 | 节点度数超过该值时默认不进入下一跳。
highDegreeLabels | string[] | 内置列表 | 自定义高出度低区分度标签。为空时使用默认列表。
keepHighDegreeNode | boolean | true | 是否保留高出度节点本身及一跳关系。
includePruningSummary | boolean | true | 是否在 summary.pruning 中返回剪枝摘要。
A.3 推荐请求示例
{ "depth": 2, "limit": 500, "traversalMode": "cascade", "prunePolicy": "degree_aware", "maxExpandDegree": 200, "keepHighDegreeNode": true, "includePruningSummary": true }
A.4 Response 剪枝摘要
两个图谱查询 API 的响应 summary 中建议包含 pruning 对象，用于说明剪枝策略、阈值、被保留为 terminal 的高出度节点数量、阻断的扩展次数以及按原因统计的剪枝数量。
字段 | 类型 | 说明
summary.pruning.policy | string | 实际使用的剪枝策略。
summary.pruning.maxExpandDegree | number | 本次请求使用的最大可扩展度数阈值。
summary.pruning.terminalHubCount | number | 被保留为 terminal 且不继续扩展的高出度节点数量。
summary.pruning.blockedExpansionCount | number | 被阻断的下一跳扩展次数。
summary.pruning.blockedByReason | object | 按 high_degree、low_signal_label、layer_budget 等原因统计。
summary.pruning.terminalHubs | array | 最多返回若干高出度 terminal 节点示例，含 id/name/degree/labels/reason。
A.5 Warning Code
Warning Code | 含义
PRUNING_APPLIED | 已启用智能剪枝策略。
HIGH_DEGREE_NODE_TERMINATED | 高出度节点已保留为一跳 terminal，但不继续进入下一跳。
LOW_SIGNAL_HUB_FILTERED | 低区分度高出度节点被过滤或仅保留摘要。
NODE_LIMIT_REACHED_AFTER_PRUNING | 智能剪枝后仍达到节点上限。
EDGE_LIMIT_REACHED_AFTER_PRUNING | 智能剪枝后仍达到关系上限。
A.6 与两个图谱 API 的关系
POST /api/v1/graph/search-all 用于从关键词找到图谱入口；POST /api/v1/graph/expand/{node_id} 用于在用户确认节点后继续展开。若两者均使用 traversalMode=cascade、prunePolicy=degree_aware、maxExpandDegree=200、相同 depth 和 limit，则返回的业务子图应保持一致或高度一致。
A.7 limit 默认值与按跳数放大规则
POST /api/v1/graph/search-all 与 POST /api/v1/graph/expand/{node_id} 的 limit 默认值均为 1000。
limit 表示单跳基础节点上限；当调用方未显式传入 nodeLimit 时，接口按 depth 自动计算有效节点上限：effectiveNodeLimit = min(limit × depth, 5000)。
默认情况下，depth=1 返回上限为 1000，depth=2 返回上限为 2000，depth=3 返回上限为 3000，依此类推；当前接口 depth 最大为 5，因此默认最高有效节点上限为 5000。
若调用方需要严格控制总节点数，可传入 nodeLimit。nodeLimit 优先级高于 limit × depth，将作为本次请求的绝对节点上限。
edgeLimit 未传入时，默认按 effectiveNodeLimit × 4 计算，且不低于 2000。
depth | limit 默认值 | 有效节点上限 | 说明
1 | 1000 | 1000 | 一跳默认最多返回 1000 个节点
2 | 1000 | 2000 | 二跳默认最多返回 2000 个节点
3 | 1000 | 3000 | 三跳默认最多返回 3000 个节点
N | 1000 | min(1000 × N, 5000) | 按跳数线性放大，并受接口最大深度约束
