# 监管违规穿透式查询系统 - 修改说明

## 修改内容

### 1. 去除 Markdown 格式输出

#### 新增功能
添加了 `remove_markdown_formatting()` 函数，用于清理所有输出中的 markdown 格式标记。

#### 修改位置
- **regulatory_query_system.py 第27-68行**: 新增 `remove_markdown_formatting()` 函数
- **regulatory_query_system.py 第173行**: 修改 `call_deepseek_api()` 方法，在返回前调用格式清理函数
- **regulatory_query_system.py 第252行**: 在生成最终答案时提示不使用 markdown 格式

#### 清理的 Markdown 元素
- 代码块标记 (```)
- 粗体标记 (**text** 或 __text__)
- 斜体标记 (*text* 或 _text_)
- 标题标记 (# ## ###)
- 链接 ([text](url))
- 图片 (![alt](url))
- 行内代码 (`code`)
- 分割线 (---)
- 列表标记 (* - +)
- 引用标记 (>)
- 多余空行

#### 效果
- **命令行输出**: 显示纯文本，无 markdown 格式
- **final_analysis.txt**: 保存纯文本分析报告
- **community_reports.txt**: 保存纯文本社区报告

### 2. 更新社区层级关系文件

#### 文件变更
- **旧文件**: community_hierarchy_v2.xlsx
- **新文件**: community_hierarchy_v3_fixed.xlsx

#### 修改位置
- **regulatory_query_system.py 第107行**:

- **test_system.py 第96行**: 同样更新测试脚本中的文件名

#### 新文件结构
community_hierarchy_v3_fixed.xlsx 包含以下列：
- source_perspective (源视角)
- source_community_id (源社区ID)
- target_perspective (目标视角)
- target_community_id (目标社区ID)
- relation_type (关系类型)
- score (得分)
- is_strong_link (是否强关联)
- evidence (证据)
- section_overlap_score (章节重叠得分)
- semantic_similarity_score (语义相似度得分)
- semantic_reasoning (语义推理)
- shared_section_count (共享章节数)
- shared_sections (共享章节)

程序只使用前7列，与旧文件兼容。

## 常见问题

### Q1: 为什么要去除 markdown 格式？
A: 使输出更适合直接阅读和处理，避免格式标记干扰内容理解。

### Q2: 新旧层级文件有什么区别？
A: 新文件包含更多元数据（得分、证据等），但程序只使用核心字段，保持向后兼容。

### Q3: 如何验证格式是否已清理？
A: 查看 final_analysis.txt，应该没有 #、**、```等 markdown 标记。
