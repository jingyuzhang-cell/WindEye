2.给定新事件，如何做穿透式查询？
基于给定的事件描述（如监管违规或违法行为），我们可以通过动态识别相关的社区结构来追踪事件的路径。具体流程如下：

* 从事件中提取关键词：调用deep seek API 从给定的事件描述中提取关键实体（如责任方、违规行为、监管机构）。

* 将关键词与社区节点匹配：这些关键词将与责任方、违规行为和监管机构社区中的节点进行匹配。可以通过提取事件中的关键词与社区标识符进行匹配来实现这一过程。

* 构建链路/路径：一旦识别出相关的社区，就可以构建“链条”关系（
  {链路文本描述，例如：
   责任方社区 #2（经营者 / 上市公司主体） 
   ↓ 
  可能导致 
  违规行为社区 #3（利用非公开信息交易、虚假承诺收益）  
  ↓ 
  触发监管 
  监管机构社区 #8（证监会及派出机构）}）：
  
  + 从责任方 → 责任义务 → 监管机构：这提供了责任方的完整义务以及它与监管机构的关系。
  
  * 从责任方 → 违法行为 → 监管机构：这揭示了责任方可能产生的违法行为，以及与监管机构的关联。
  
  * 对路径进行可视化网络可视化：
  
    可以使用networkx或igraph等库来可视化这些链条，通过绘制图表来展示不同社区（节点）通过关系（边）连接的方式，从而展示责任方、违规行为和监管机构之间的完整路径。
  
    可视化图将包括表示不同实体（责任方、违规行为、监管机构）的节点，以及它们之间的关系（边），展示责任方与违规行为、监管机构之间的完整连接路径。

+ 把“链路中节点的所在社区报告”直接加入提示词（prompt），作为主要上下文；引导deep seek结合知识图谱与所在链路上的社区报告生成最终答案。

5. 系统工作流程

* 步骤1：事件分析：对于给定事件，提取相关信息（如责任方、监管机构、违规行为）。

* 步骤2：追踪路径：使用识别出的实体，追踪事件相关的社区（责任方、监管机构、违规行为）。

* 步骤3：生成报告：使用DeepSeek API生成每个社区视角的社区报告，包含完整的法律内容和发现。

* 步骤4：可视化与总结：创建可视化图，展示事件与法规/违规结构之间的关系，并生成汇总表格或Excel文件进行报告。
