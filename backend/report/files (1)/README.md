# 监管违规穿透式查询系统 - 使用说明

## 系统概述

本系统基于社区检测和知识图谱技术，实现对监管违规事件的穿透式分析。系统能够：
1. 从事件描述中自动提取关键实体（责任方、违规行为、监管机构）
2. 将实体与预构建的社区结构进行匹配
3. 基于社区层级关系构建事件链路
4. 生成可视化网络图
5. 结合社区报告生成综合分析

## 系统架构

### 数据文件说明

1. **社区可视化数据** (JSON格式)
   - `regulatory_visualization_data.json`: 监管机构视角的社区结构
   - `responsibility_visualization_data.json`: 责任方视角的社区结构
   - `violation_visualization_data.json`: 违规行为视角的社区结构

2. **社区报告** (Excel格式)
   - `监管机构社区报告.xlsx`: 包含每个监管机构社区的详细报告
   - `责任方社区报告.xlsx`: 包含每个责任方社区的详细报告
   - `违规行为社区报告.xlsx`: 包含每个违规行为社区的详细报告

3. **社区关系**
   - `community_hierarchy_v2.xlsx`: 预定义的社区间关联关系

4. **知识图谱**
   - `merged_regulatory_unified.txt`: 从资本市场法规中提取的实体和关系

## 安装依赖

```bash
pip install pandas openpyxl networkx matplotlib requests --break-system-packages
```

## 配置API密钥

在运行系统前，需要配置DeepSeek API密钥。有两种方式：

### 方式1: 修改程序文件
编辑 `regulatory_query_system.py`，找到第17行：
```python
API_KEY = ""  # 用户需要填入API密钥
```
将您的API密钥填入引号中。

### 方式2: 修改配置文件
编辑 `config.py`：
```python
API_KEY = "your-api-key-here"
```

## 运行系统

### 基本使用

```bash
python3 regulatory_query_system.py
```

### 输入事件描述

系统启动后会提示输入事件描述：
```
请输入事件描述（输入完成后按Enter，再输入'END'并按Enter结束）：
```

您可以输入多行事件描述，例如：
```
某上市公司在未披露重大信息的情况下，
实际控制人利用内幕信息进行股票交易，
涉嫌内幕交易和信息披露违规。
END
```

## 工作流程

### 步骤1: 事件分析
系统调用DeepSeek API从事件描述中提取：
- 责任方实体（如：上市公司、实际控制人）
- 违规行为（如：内幕交易、信息披露违规）
- 监管机构（如：证监会）

### 步骤2: 社区匹配
系统将提取的实体与三种视角的社区节点进行匹配：
- 责任方社区：找出相关的责任主体社区
- 违规行为社区：识别涉及的违规类型社区
- 监管机构社区：确定相关的监管机构社区

### 步骤3: 路径构建
基于 `community_hierarchy_v2.xlsx` 中预定义的社区关系，构建从责任方到违规行为再到监管机构的完整路径。

### 步骤4: 网络可视化
生成可视化网络图，展示：
- 不同颜色表示不同视角的社区
  - 橙色：责任方社区
  - 红色：违规行为社区
  - 蓝色：监管机构社区
- 箭头表示社区间的关联关系
- 线条粗细表示关联强度

### 步骤5: 报告生成
系统生成三类报告：

1. **社区报告** (`community_reports.txt`)
   - 包含所有匹配社区的详细报告
   - 包括社区标题、摘要、关键词、主要发现

2. **综合分析报告** (`final_analysis.txt`)
   - DeepSeek基于社区报告和知识图谱生成的深度分析
   - 包含责任分析、违规认定、监管依据、法律后果等

3. **Excel汇总表** (`analysis_summary.xlsx`)
   - 事件概述
   - 匹配的社区列表
   - 路径分析表

## 输出文件

所有输出文件保存在 `/mnt/user-data/outputs/` 目录：

1. `event_network.png` - 事件网络可视化图
2. `community_reports.txt` - 社区报告汇总
3. `final_analysis.txt` - 综合分析报告
4. `analysis_summary.xlsx` - Excel汇总表

## 系统特点

### 1. 自动化实体提取
利用大语言模型自动识别事件中的关键信息，无需手动标注。

### 2. 多视角分析
从责任方、违规行为、监管机构三个维度全面分析事件。

### 3. 知识图谱支持
基于真实法规数据构建的知识图谱，确保分析的法律依据准确。

### 4. 可视化呈现
直观的网络图展示事件涉及的各方关系和法律链路。

### 5. 深度分析报告
结合社区报告和AI分析，生成专业的综合评估。

## 故障排除

### 问题1: API调用失败
- 检查API_KEY是否正确配置
- 确认网络连接正常
- 检查DeepSeek账户余额

### 问题2: 未匹配到社区
- 事件描述可能过于简单或模糊
- 尝试使用更具体的实体名称
- 系统会自动启用关键词模糊匹配

### 问题3: 缺少依赖包
```bash
pip install pandas openpyxl networkx matplotlib requests --break-system-packages
```

### 问题4: 中文显示问题
- 确保系统安装了中文字体
- Linux系统可能需要安装：`sudo apt-get install fonts-wqy-zenhei`

## 技术细节

### 社区检测算法
系统使用Louvain算法对知识图谱进行社区检测，将相关实体聚类到同一社区。

### 匹配策略
- 精确匹配：实体名称完全匹配节点名称
- 包含匹配：实体名称包含或被包含于节点名称
- 模糊匹配：基于关键词的相似度匹配

### API调用优化
- 使用较低的temperature（0.3）确保输出稳定
- 限制max_tokens避免超时
- 异常处理确保系统鲁棒性
