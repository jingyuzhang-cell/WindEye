# 多视角社区层级构建方案 - 使用说明

## 📋 项目概述

本项目实现了基于三个视角（责任方、违规行为、监管机构）的社区层级构建方案，通过计算社区间的连接强度，构建了一个多层级的关系网络。

## 🎯 核心功能

### 1. 层级定义
- **L0（最上层）**: 责任方视角社区
- **L1（中间层）**: 违规行为视角社区  
- **L2（最下层）**: 监管机构视角社区

### 2. 连接强度计算

连接强度基于四个维度综合评分：

```
连接强度 = 
  0.50 × Section重叠比例 +
  0.20 × 核心节点Jaccard相似度 +
  0.15 × 高权重跨边密度 +
  0.15 × 语义相似度
```

#### 各维度说明：

**1) Section重叠比例（权重：0.50）**
- 计算方式：containment（交集 / 较小集合大小）
- 业务含义：法规条款是最强的逻辑锚点
- 必须性：必须

**2) 核心节点重叠（权重：0.20）**
- 计算方式：Jaccard相似度
- 关注节点：PartyWithResponsibility, AdvantageHolder, Actor等
- 业务含义：主体类型一致性
- 必须性：强烈推荐

**3) 跨社区高权重关系密度（权重：0.15）**
- 计算方式：权重≥0.7的跨边数量比例
- 业务含义：强关系连接程度
- 必须性：推荐

**4) 语义相似度（权重：0.15）**
- 计算方式：核心节点类型重叠
- 业务含义：语义合理性
- 必须性：可选

### 3. 连接阈值

| 连接类型 | 最低分数阈值 | 允许关系 | 连接强度 |
|---------|------------|---------|---------|
| 责任方 → 违规行为 | 0.50 | 多对多 | 强连接 |
| 违规行为 → 监管机构 | 0.45 | 多对多 | 强连接 |
| 责任方 → 监管机构 | 0.40 | 多对多 | 弱连接 |

## 🔧 技术实现

### 核心算法

```python
# 1. Section重叠计算
def calculate_section_overlap(sections1, sections2):
    intersection = len(sections1 & sections2)
    smaller_size = min(len(sections1), len(sections2))
    return intersection / smaller_size if smaller_size > 0 else 0.0

# 2. Jaccard相似度
def calculate_jaccard(set1, set2):
    intersection = len(set1 & set2)
    union = len(set1 | set2)
    return intersection / union if union > 0 else 0.0

# 3. 跨边密度
def calculate_cross_edge_density(source_nodes, target_nodes, edges):
    high_weight_edges = [e for e in edges 
                        if e['source'] in source_nodes 
                        and e['target'] in target_nodes 
                        and e['weight'] >= 0.7]
    total_out_edges = [e for e in edges if e['source'] in source_nodes]
    return len(high_weight_edges) / len(total_out_edges)
```

def find_path(source_comm, target_comm):
    # 责任方 → 违规行为
    middle_comms = find_connections('responsibility', source_comm, 'violation')
    
    # 违规行为 → 监管机构
    for middle in middle_comms:
        end_comms = find_connections('violation', middle, 'regulatory')
        if target_comm in end_comms:
            return f"R{source_comm} → V{middle} → G{target_comm}"
