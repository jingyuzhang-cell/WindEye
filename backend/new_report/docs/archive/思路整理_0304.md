在已有**三视角社区层级结构**的基础上，实现：

“当用户给出一段未定性的资本市场事件描述时，既能给出**宏观社区责任递进链**（责任方社区 → 违规行为社区 → 监管机构社区），又能进一步钻取到**具体节点之间的证据路径**，并可视化展示这两层信息。”

阶段1. 输入事件描述 → 粗粒度社区激活（利用已有层级结构）
   ↓
阶段2. 在“被激活的社区”内部 + 跨社区边界边 上进行 细粒度节点召回
   ↓
阶段3. 构建跨视角“小规模证据子图”（30–150节点）
   ↓
阶段4. 在证据子图上做约束路径搜索（带社区流向偏置 + 关系模板偏置）
   ↓
阶段5. 输出双层结果 + 双层可视化
   ├── 层1（宏观）：社区级责任递进路径（已有结果直接用）
   └── 层2（微观）：节点级证据路径（新计算）



## 各阶段简单解释：

### 阶段1：先粗略判断“大概跟哪几个社区有关”（社区级粗匹配）

系统现在不会直接去看几千个节点，它先做一道“筛选题”：

1. 用大模型读这句话，圈出关键名词和动作  
   → 提取出：实际控制人、并购消息、二级市场买入、内幕交易

2. 把这些词去跟三个视角（责任方 / 违规行为 / 监管机构）的所有社区名、社区关键词、社区报告摘要去比对  
   （这里大量利用你之前已经做好的社区报告和社区之间的层级关系）

3. 发现匹配度比较高的社区，比如：
   - 责任方视角：社区#4（大股东、实际控制人、高管）
   - 违规行为视角：社区#9（内幕信息交易、短线交易、利用非公开信息）
   - 监管机构视角：社区#2（证监会稽查、行政处罚）

4. 再看你之前已经算好的社区层级关系图，发现正好有一条现成的路径：  
   责任方社区#4 → 违规行为社区#9 → 监管机构社区#2

→ **这一步的产出**：系统先锁定了“大概率跟这三个社区有关”，而且大概率是沿着“责任方→违规→监管”这个大方向走。

### 阶段2：在“已经锁定的几个社区”里面，继续找最像的那些具体节点

现在系统把搜索范围大幅缩小，只看上面三个社区里的节点（而不是全图几千个节点）。

它会做两件事：

- A. 直接字符串匹配 + 模糊匹配  
  例如把“实际控制人”“并购”“内幕交易”这些词，跟社区里的节点名字比对，谁名字里包含这些词，谁就被优先选出来。

- B. 把用户这句话整体拿去跟社区里每个节点的描述（名字+属性+所在条款片段）做语义相似度比对  
  （可以用文字向量化模型，例如 bge-small-zh，把句子转成向量，然后算距离，距离越近越相关）

假设每个社区最后挑出 8–15 个最像的节点，三个社区加起来大概 30–45 个核心节点。

→ **这一步的产出**：得到一份“嫌疑最大”的节点名单，例如：
- 实际控制人
- 非公开信息
- 并购事项
- 内幕交易
- 短线交易
- 《证券法》第52条
- 证监会行政处罚
- ……

### 阶段3：把这些嫌疑节点周围的关系拉出来，拼成一张“小图”

系统不再看全图，而是以刚刚选出的 30–45 个节点为核心，向外“抠”一圈关系：

- 每个节点拉出它的直接出边（它指向谁）和直接入边（谁指向它） → 通常 1跳
- 如果跨了社区，但这条跨社区边在你之前的层级关系里被标记为“强连接”，也允许保留

结果大概会得到 60–150 个节点、200–500 条边的一张“小证据图”。  
这张图已经比全图小很多，但仍然包含了跟用户问题最可能相关的所有实体和关系。

→ **这一步的产出**：一张规模可控的“小知识图谱切片”

### 阶段4：在小图上找“最能解释这件事的几条路”

现在系统在这张小图上专门找路径，找的路径要尽量符合下面几个倾向：

- 起点尽量在“责任方视角”的社区
- 中间尽量经过“违规行为视角”的社区
- 终点尽量到达“监管机构视角”的社区
- 优先走那些关键的关系类型，例如：知悉 → 利用 → 实施 → 违反 → 处以 → 处罚

系统可能会用几种方式找路径：

- 方法A：直接找从责任方节点到监管节点的所有短路径（3–6 步那种）
- 方法B：给每种关系打一个“重要加分”，例如“实施”“违反”“处以”这些词加很多分，然后用类似导航软件的“A*算法”找最高分的路径
- 方法C：跑 100 次带偏置的随机游走，看哪几条路被走得最多

最后选出得分最高的前 3–5 条路径，作为“证据链”。

→ **这一步的产出**：几条具体的链条，例如  
实际控制人 → 获知 → 非公开并购信息 → 指使 → 亲属买入 → 构成 → 内幕交易 → 违反 → 证券法第52条 → 证监会 → 立案调查并处罚

### 阶段5：把宏观 + 微观一起给用户看

最终呈现给用户的内容分成两层：

**第一层（给领导/非专业人士看的宏观结论）**  
用你之前已经画好的社区层级图，直接标红那条路径：  
责任方社区#4 → 违规行为社区#9 → 监管机构社区#2

**第二层（给办案人员/律师看的微观证据）**  
把阶段4找到的那几条具体节点路径列出来 + 画一张小图  
（小图上用不同颜色标注不同视角的节点，路径用粗线高亮）

然后把这两层内容一起塞进 prompt，让大模型写一份完整的分析报告。

prompt推荐内容为“
你现在是一位非常专业的资本市场合规律师。

已知事实：
{用户输入的事件描述}

已提取的关键实体：{提取出的实体列表}

从知识图谱中召回的相关事实片段（已按相关性排序）：
{把搜到的路径或高相关三元组/短路径按重要性列出，每条前加序号}

更广的上下文（社区级摘要，仅供参考）：
{原来三个视角最相关社区的 summary + key findings}

请严格按照以下结构回答，不得遗漏任何一项：

1. 初步定性结论（一句话）
2. 核心事实依据（引用图谱中的具体节点和关系，用编号对应上面的事实片段）
3. 涉及的主要法规条款（尽量写出条款号）
4. 可能适用的具体罪名 / 违法情形 / 违规情形（列出最匹配的2~4项）
5. 可能的法律后果区间（行政 / 民事 / 刑事）
6. 对当事人的合规建议（3~6条可执行建议）”

### 目前最务实、可快速验证的组合（建议 1～2 周内跑出 demo）

1. **社区粗筛**：用已有社区层级 + 简单字符串匹配 + LLM 实体提取
2. **节点召回**：每个被激活社区内做 **BM25** 或 **fuzzy match** 取 Top-10 节点
3. **子图构建**：1跳出边 + 1跳入边 + 允许跨社区强连接边
4. **路径搜索**：
   - 用 networkx dijkstra_path 或 astar_path
   - 自定义 cost = 1 / edge_weight
   - 额外加 community_flow_bonus（责任→违规 +0.4，违规→监管 +0.5 等）
5. **可视化**：用 pyvis 生成一张交互式 HTML 图（节点带颜色表示视角，边带粗细表示权重）
