### 一.构建**多视角社区映射结构**

#### 1.层级定义

层级0（最上层）：责任方社区
层级1（中间层）：违规行为社区
层级2（最下层）：监管机构社区

形成带有方向性的社区对应关系。允许存在多父、多子、孤立社区、两层结构、三层结构

#### 2.连接关系的评分依据（2种信号综合打分）

1）Section / Law 重叠比例

+ 计算方式：计算章节、条款的重叠比例（责任方社区包含的条款集合 → 违规行为社区包含的条款集合）

+ 含义：条款是法规逻辑最强的锚点

+ 权重：0.50

 2）社区报告语义相似度

+ 计算方式：将两个社区的 summary / findings / key_words 喂给 deep seek，打分（0-10分）
+ 含义：语义上是否合理构成“因→果”或“主体→行为”关系
+ 权重：0.50

#### 3.连接强度计算

连接强度 =  0.50 × containment(责任方Section集合, 违规行为Section集合) +  0.50 × (LLM语义相似度分值 / 10)

| 层间连接            | 阈值                    |                                        |
| ------------------- | ----------------------- | -------------------------------------- |
| 责任方 → 违规行为   | 最低分数阈值：0.45–0.55 | 允许多对多                             |
| 违规行为 → 监管机构 | 最低分数阈值：0.40–0.50 | 允许多对多                             |
| 责任方 → 监管机构   | 最低分数阈值：0.35–0.45 | 允许（作为跳跃式补充，两层结构时有用） |

孤立社区与弱连接处理
- 如果一个社区没有任何得分超过阈值的连接，则作为孤立社区单独存在
- 弱连接（分数低于主干阈值但仍有一定分数）可标记为“建议连接”或“可能相关”，不作为主干路径
