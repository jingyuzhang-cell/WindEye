通过多视角社区层级构建方案以“责任方”作为最上层，向下连接违规行为社区（最常见的中间层），再向下连接监管机构社区（最底层的执法主体）实现社区的对应关系树（允许存在多父、多子、孤立社区、两层结构、三层结构），从而实现多视角社区层级构建

多视角社区层级构建方案：
（以“责任方”作为最上层，向下连接违规行为社区，再向下连接监管机构社区）

一.目标
以责任方视角社区作为层级最上层（L0），向下连接违规行为视角社区（L1，最常见中间层），再向下连接监管机构视角社区（L2，最底层执法主体），形成带有方向性和层级感的社区对应关系树。
允许结构特点：多父、多子、孤立社区、两层结构、三层结构。

二.核心原则
层级顺序采用业务先验硬编码，而不是完全数据驱动。
责任方 → 违规行为 → 监管机构 的顺序作为固定约束。

三.层级定义
层级0（最上层）：责任方视角的所有社区
层级1（中间层）：违规行为视角的所有社区
层级2（最下层）：监管机构视角的所有社区

连接关系的判断依据（四种信号综合打分）
1. Section / Law 重叠比例
   - 计算方式：containment 或 Jaccard（责任方社区包含的条款集合 → 违规行为社区包含的条款集合）
   - 业务含义：条款是法规逻辑最强的锚点
   - 建议权重：0.50
   - 是否必须：必须

2. 核心节点重叠比例
   - 计算方式：Jaccard（责任方社区的核心节点集合 与 违规行为社区的核心节点集合）
   - 核心节点主要指：PartyWithResponsibility, AdvantageHolder, Actor 等
   - 业务含义：主体类型是否高度一致
   - 建议权重：0.20
   - 是否必须：强烈推荐

3. 跨社区高权重关系密度
   - 计算方式：从责任方社区节点指向违规行为社区节点的高权重边（weight ≥ 0.7）的数量或比例
   - 业务含义：是否存在明确的“导致”“执行”“产生”等强关系
   - 建议权重：0.15
   - 是否必须：推荐

4. 社区报告语义相似度（可选）
   - 计算方式：将两个社区的 summary / findings / key_words 喂给 LLM，打分（0-10分）
   - 业务含义：语义上是否合理构成“因→果”或“主体→行为”关系
   - 建议权重：0.15
   - 是否必须：可选

推荐打分公式（可调参）
连接强度 = 
  0.50 × containment(责任方Section集合, 违规行为Section集合) +
  0.20 × jaccard(责任方核心节点, 违规行为核心节点) +
  0.15 × (责任方→违规行为的高权重跨边数量 / 责任方社区出边总数) +
  0.15 × (LLM语义相似度分值 / 10)

层间连接阈值建议
责任方 → 违规行为     最低分数阈值：0.45–0.55    允许多对多
违规行为 → 监管机构   最低分数阈值：0.40–0.50    允许多对多
责任方 → 监管机构     最低分数阈值：0.35–0.45    允许（作为跳跃式补充，两层结构时有用）

孤立社区与弱连接处理
- 如果一个社区没有任何得分超过阈值的连接，则作为孤立社区单独存在
- 弱连接（分数低于主干阈值但仍有一定分数）可标记为“建议连接”或“可能相关”，不作为主干路径
