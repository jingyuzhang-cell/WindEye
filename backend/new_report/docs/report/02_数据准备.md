# 数据准备

## 输入文件

主输入文件默认是：

- `D:\Code\WindEye\backend\new_report\code\data\merged_regulatory_unified.txt`

它是 JSONL 格式的统一知识图谱文本，包含：

- `type=node` 的节点记录
- `type=relationship` 的关系记录

辅助输入目录：

- `backend/new_report/code/policy_docs`：政策法规原文 TXT
- `backend/new_report/code/data/关系类型及权重.xlsx`：关系权重参考

## 环境配置

配置统一放在 [backend/.env](/D:/Code/WindEye/backend/.env:1)：

- `DEEPSEEK_API_KEY`：离线报告与层级映射使用的 LLM Key

如果没有显式设置 `DEEPSEEK_API_KEY`，系统会回退使用 `LLM_API_KEY`。

## 输出目录约定

所有新生成产物默认写入：

- `backend/report_outputs/gnn_leiden_results`
- `backend/report_outputs/community_reports`
- `backend/report_outputs/build_hierarchy_links_output`

这些路径由代码按当前仓库根目录自动推导，不需要用户在 `.env` 里额外配置。

## 数据准备建议

- 保持 `merged_regulatory_unified.txt` 中节点和关系字段命名稳定
- 政策原文尽量使用 UTF-8 编码 TXT
- 新增样本时优先补充法规原文，再运行 Step2 与 Step3
