# 系统概述

## 定位

`backend/new_report` 是一套离线批处理的多视角监管合规社区分析系统。它面向深度分析场景，不承担在线接口的实时社区发现职责。

与 [governance_routes.py](/D:/Code/WindEye/backend/api/governance_routes.py:251) 的在线社区发现接口不同，这套系统采用预计算模式，先离线聚类、再离线生成社区报告、最后离线完成跨视角层级映射。

## 核心思路

系统将 Neo4j 导出的统一知识图谱文本按三种业务视角分别切分和聚类：

- 责任方视角
- 违规行为视角
- 监管机构视角

在每个视角下生成社区结果后，再通过法规条文重叠和语义相似度建立跨视角因果链，最终支撑“双层穿透式查询”：

- 宏观层：社区链路和风险传播结构
- 微观层：节点证据链和原始关系依据

## 目录结构

- `backend/new_report/code`：主流程代码、独立算法、历史版本、样例输入
- `backend/new_report/docs/report`：当前维护文档
- `backend/new_report/docs/archive`：历史思路与方案文档
- `backend/new_report/tests`：校验脚本和历史测试
- `backend/report_outputs`：统一生成产物目录
