# 双层穿透式查询

## 目标

双层穿透式查询不是一个单独的离线步骤，而是建立在社区聚类、社区报告和层级映射之上的查询能力。

## 查询链路

推荐分成五个阶段：

1. 社区粗筛：先锁定最相关的责任方/违规行为/监管机构社区
2. 节点召回：从候选社区中召回核心节点、行为节点和法规节点
3. 子图构建：围绕问题意图抽取局部证据子图
4. 约束路径：在候选路径中保留满足法规与时序约束的路径
5. 双层输出：同时返回宏观社区链和微观节点证据链

## 与在线接口的关系

在线接口可以消费离线社区报告作为补充上下文，但不应直接复用历史原型脚本。更合适的方式是读取 `backend/report_outputs` 中的稳定产物，再做线上适配。
